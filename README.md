# MC Apply Bot

Telegram-бот для приёма заявок на участие в Minecraft-сервере.

## Возможности

- Игроки подают заявку через пошаговый диалог: ник в Minecraft → опыт игры → планы на
  сервере → ссылки на видео → подтверждение.
- Заявки автоматически уходят администраторам (в группу или в личку каждому админу)
  с кнопками **✅ Одобрить** / **❌ Отклонить**.
- При отклонении админ может либо отклонить сразу, либо написать причину — она придёт игроку.
- Бот различает админов (список `ADMIN_IDS` в `.env`) и обычных пользователей — кнопки
  решений видны всем в чате, но нажать их может только админ (иначе бот покажет
  всплывающее уведомление "нет прав").
- Игрок может посмотреть статус своих заявок командой `/myapplications`.
- Заявки хранятся в SQLite (`applications.db`), при перезапуске бота ничего не теряется.

## Установка

1. Убедитесь, что установлен Python 3.11+.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Получите токен бота у [@BotFather](https://t.me/BotFather) (команда `/newbot`).
4. Узнайте свой Telegram user_id, например у [@userinfobot](https://t.me/userinfobot).
5. Скопируйте `.env.example` в `.env`:
   ```bash
   cp .env.example .env
   ```
6. Заполните `.env`:
   - `BOT_TOKEN` — токен от BotFather
   - `ADMIN_IDS` — ваш user_id и id остальных админов через запятую
   - `ADMIN_CHAT_ID` — (по желанию) id группы для приёма заявок

### Если хотите слать заявки в группу админов

1. Создайте группу в Telegram, добавьте туда бота.
2. Добавьте в группу [@RawDataBot](https://t.me/RawDataBot) — он пришлёт `chat.id` группы
   (число со знаком минус, например `-1001234567890`).
3. Впишите это значение в `ADMIN_CHAT_ID` в `.env`.
4. Убедитесь, что у бота в группе есть право отправлять сообщения.

Важно: даже при отправке в общую группу, нажать кнопки "Одобрить"/"Отклонить" сможет
только тот, чей user_id указан в `ADMIN_IDS` — остальные участники группы увидят
уведомление об отсутствии прав.

## Запуск

```bash
python bot.py
```

Бот запустится в режиме polling. Для продакшена рекомендуется запускать через
`systemd`/`supervisor`/`screen`/`tmux`, чтобы процесс не останавливался при закрытии терминала.

## Команды бота

**Для всех:**
- `/start` — приветствие и кнопка подачи заявки
- `/apply` — начать заполнение анкеты
- `/cancel` — отменить заполнение анкеты
- `/myapplications` — статус своих заявок

**Только для админов (`ADMIN_IDS`):**
- `/pending` — список всех заявок на рассмотрении с кнопками
- `/cancel_reject` — отменить ввод причины отказа, если передумали

## Возможные доработки

- Хранить видео как `MessageHandler(filters.VIDEO | filters.TEXT)`, если хотите принимать
  видео-файлы напрямую, а не только ссылки.
- Добавить лимит на количество заявок в день от одного пользователя.
- Добавить экспорт одобренных заявок в CSV/Excel для вайтлиста сервера.
- Подключить вебхуки вместо polling для продакшн-нагрузки.
